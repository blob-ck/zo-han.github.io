HEXO + Github 블로그 간단 정리

2018-10-04
hexo

Hexo 요약

사전준비

  1. Node.js 설치
  2. Git 설치
  3. Github repository 생성

    유저명.github.io 로 repository 생성

HEXO

  1. hexo 설치(어디서든)
    > npm install -g hexo-cli
     

  2. hexo 로 블로그 기반 폴더 초기화(사용할 폴더의 상위폴더에서)
    상위폴더> hexo init 폴더명
     

  3. 폴더 이동
    상위폴더> cd 폴더명
     

  4. Node Package Manager 로 hexo에 필요한 것 설치
    블로그> npm install
     

  5. Hexo 테마 설치 (hueman 테마 - 블로그/themes/hueman 에 복사된다.)
    블로그> git clone https://github.com/ppoffice/hexo-theme-hueman.git themes/hueman
     

  6. 블로그/_config.yml 을 열어 수정한다. (themes/hueman/_config.yml 과는 다름)
    theme: hueman #기본테마는 landscape로 되어있다.
     

  7. 블로그/themes/hueman/_config.yml.example 파일명 변경
    _config.yml.example -> _config.yml
     

  8. github에 배포할 것이므로 deploye-git 설치
    블로그> npm install hexo-deployer-git --save
     

  9. hexo 정적 블로그 생성(html, css, js 등등)
    블로그> hexo generate
     

  10. 깃헙에 배포하기 전에 로컬에서 테스트 하려면(생략가능)
    hexo server
    http://localhost:4000 으로 접속하여 테스트
     

  11. Github에 배포
    블로그> hexo deploy
     

  12. 6+7 위 두 줄을 한번에 하려면
    블로그> hexo g -d
     

while(살아있음) {

}

 
 

카테고리 설정

포스트 작성시 헤더 부분 (테마별로 다를 수 있다. 포스팅 썸네일도 추가할 수 있다.)

이렇게 포스트 작성하여 생성후 배포(hexo g -d)하면
아래처럼 블로그의 카테고리가 만들어진다.

 
 

==주의사항==

hexo 테마의 경우 깃헙에 배포를 해도 정적파일만 생성해서 올릴 뿐이므로,
블로그 테마의 설정은 올라가지 않는다.
따로 저장소를 추가로 만들어 블로그 설정을 백업하는 것이 좋다.
(타 블로그에서 hexo 백업 검색하면 submodule 사용하는 방법도 나온다.)
jekyll때는 그런거 없이 그냥 다 올라가서 폴더구조가 더 헷갈렸던것 같기도 하고…
나중에 귀찮아서 jekyll로 갈아탈지도

 
 
 

카테고리에 대한 잡설
네이버 블로그를 만들었을 때는 카테고리 항목을 미리 만들어놓고
그에 맞는 포스트를 작성했었는데,
구글 블로거 라든지 외국 블로그는 포스팅에 카테고리를 저장해야 만들어진다.
빈 카테고리를 용납하지 않는건지…
조승연 작가의 책 중 ‘플루언트’가 생각난다.
동양은 수렴형 사고, 서양은 발산형 사고
전체에서 나 로 수렴하는 동양은 제일 큰 범주부터 생각을 하고,
서양은 나로부터 뻗어나가는 사고방식의 차이?

Think DataStructure with Java

2018-10-23
til, ds

Think-DataStructure-with-Java

얇다.
익숙한 자바로 자료구조를 익힐 수 있다.
어떤 경우에 적용할 지 연상할 수 있다.

부가-intelli-j-용-소스도-있다.

일단 시작하기 좋고 더 깊게 가기 전 마중물로 충분해 보였기에 바로 구입하여 날마다 기록할 예정.
단, 실습에 앞서 선지식을 요구하는데

  • ant - 아파치 재단에서 만든 자바 자동화 빌드 도구
  • JUnit - 자바를 위한 유닛 테스트 프레임워크
  • 자바 상속, 인터페이스, 타입 파라미터와 제네릭 타입, 컬렉션 등

이 중 ant를 써본 경험이 없어 일단 maven이나 gradle을 써봤으므로
이 전신인 ant도 비슷할 것이라는 막무가내 추측으로 (사실 구 버전이고 학습의 주 는 자료구조이므로)
scoop install antbuild.xml 파일이 있는 곳으로 이동하여ant 실행소스명치니까 어? 되네? …
일단은 이렇게 빌드파일이 있는 곳으로 가서 ant 이름 치면 성공하는 것을 확인했으므로,오케이.

뭐 소스에서 java application으로 실행해도 문제는 없으니 넘어간다.

리스트가 두 종류인 이유 - 앞으로 몇 장에 걸쳐 알아봄interface와 이를 구현하는 클래스를 살피고 인터페이스 프로그래밍의 개념을 알아본다.
각기 다른 리스트구현클래스 - LinkedList 와 ArrayList 의 쓰임새(장단점)자바에서 interface는 메소드 집합을 의미한다.
interface를 구현하는 class는 이 메소드 집합을 구현해야 한다.
(이클립스에서 interface를 implements 하면 자동으로 Add unimplemented methods 가 뜨는 이유)

알고리즘 분석

  • 프로파일링 : 어떤 방법이 더 좋은지 적용해보고 판단하는 방법
  • 알고리즘 분석 : 실제 적용하여 판단하기보다, 몇 가지 가정 하에 예상하여 판단하는 방법

경제학원론을 쓴 마셜? 의 세테리스 파리부스 == ‘다른 모든 조건이 동일하다면’
필수 요소들만으로 모형을 만들어 분석하는 방법이 어렴풋이 떠올랐다.
빅오말 그대로 영향력이 가장 큰 변수 하나만을 대표로 분석하는 방법으로

  • 상수
  • 선형
  • n^2
    등으로 큼직하게 구분한다. (실행수를 무한대로 하면 더욱 가까워지겠지)
    선택정렬을 일단 구현하고,다시 처음부터 구현하면서 모듈화 할 부분을 나눈다.
    조금씩 만들어가며 junit으로 단위테스트를 한다.최종 테스트까지 오케.
    코딩골프라는 책을 잠깐 봤는데 이렇게 잘게잘게 쪼게어 객체지향으로 유지보수 좋게 하는 방법과는
    정 반대로 대치되는 것 같다.
    어떻게든 적은 소스로 보다 효율적으로 보다 빠르게 가 목적인 만큼,재미는 있을 듯.
